---
title: Run an evidence council
description: Combine architecture, UX, research, QA, and security perspectives without letting reviewers directly rewrite authoritative artifacts.
---

## Define the question

Provide one decision or artifact boundary, required lenses, available evidence, and the owner who can accept proposals. A council without a bounded question produces commentary, not a review.

## Choose execution mode

Use simulated mode for a portable single-assistant review. Use independent mode only when the host can run genuinely separate reviewers with isolated context. Label the mode honestly.

## Compare findings

Require each lens to cite evidence, state confidence, and distinguish facts from proposals. The council report should show agreements, conflicts, unresolved questions, owners, and next actions.

## Protect authority

Council participants do not edit requirements, design, policy, or release approval directly. The artifact owner reviews proposals and records accepted changes through the normal decision and change-impact workflow.
