# Guides

This folder explains how PM, BA, QA, and Dev roles work with the AI SDLC skill
library.

Guides describe role and workflow behavior:

- [Workflow](workflow.md) explains the overall AI-ready delivery model.
- [PM](pm.md) explains product discovery, goals, PRFAQ, backlog, and release
  shaping.
- [BA](ba.md) explains context engineering, business rules, acceptance criteria,
  and delivery refinement.
- [QA](qa.md) explains QA-first planning, test design, validation, and
  traceability.
- [Dev](dev.md) explains SDD, implementation validation, code review, and commit
  readiness.

Technical mechanics such as artifact routing, flow modes, decision logs, state
machines, metadata, specs indexes, scripts, and traceability live in
`concepts/`.
